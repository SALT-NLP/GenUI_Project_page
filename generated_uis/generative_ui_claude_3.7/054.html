React Native Ecosystem

What is React Native?

React Native is a framework for building native mobile applications using JavaScript and React. It allows you to create mobile apps that can run on both iOS and Android platforms using a single codebase.

For Web Developers

If you're coming from web development, think of React Native as React for mobile devices. Many concepts you already know from React will transfer directly, but instead of rendering to the DOM, React Native renders to native mobile components.

Key Components in React Native

Core Components

React Native provides a set of built-in components that map directly to native UI elements on iOS and Android. These replace the HTML elements you're used to in web development.

Web Element React Native Component Description
<div> <View> A container component for other components
<span>, <p>, <h1> <Text> Displays text (all text must be inside a Text component)
<img> <Image> Displays images from various sources
<input>, <textarea> <TextInput> Text input fields
<button> <Button>, <Pressable>, <TouchableOpacity> Interactive elements for user input
<ul>, <ol> <FlatList>, <SectionList> Optimized lists for displaying large amounts of data
<a> <Linking> (API) Opening URLs, not a component but an API
<form> No direct equivalent Forms are created using combinations of other components

Styling in React Native

Unlike web development where you use CSS, React Native uses JavaScript objects for styling that closely resemble CSS-in-JS syntax.

Example: Styling in React Native
Key Differences in Styling
  • All dimensions are unitless and represent density-independent pixels
  • Styles are not inherited (except for text within Text components)
  • Flexbox is the primary layout system, but with some differences from web
  • No CSS Grid support
  • Property names are camelCase (e.g., backgroundColor instead of background-color)
  • No cascading - styles don't automatically apply to child components

Development Workflow

Setting Up Your Environment

There are two main approaches to starting a React Native project:

Expo CLI (Recommended for Beginners)

Expo provides a managed development environment with a simplified workflow and access to many native features without writing native code.

Setting up with Expo 
# Install Expo CLI
npm install -g expo-cli

# Create a new project
expo init MyNewProject

# Start the development server
cd MyNewProject
expo start
Expo Advantages
  • Faster setup - no need to install Xcode or Android Studio initially
  • Expo Go app lets you test on physical devices by scanning a QR code
  • Built-in access to many native features via Expo SDK
  • Over-the-air updates without app store approval
  • Simplified build process for app stores

React Native CLI

The React Native CLI provides more control and direct access to native code, but requires more setup and maintenance.

Setting up with React Native CLI 
# Install React Native CLI
npm install -g react-native-cli

# Create a new project
npx react-native init MyNewProject

# Start the development server
cd MyNewProject
npx react-native start

# Run on Android or iOS
npx react-native run-android
# or
npx react-native run-ios
React Native CLI Advantages
  • Full access to native code (iOS/Android)
  • No limitations on which native modules you can use
  • More control over the build process
  • Better for projects that need custom native functionality

The React Native Bridge

A key concept in React Native is the "bridge" that connects JavaScript code to native components. Your JavaScript code runs in a JavaScript thread, and communicates with the native modules via this bridge. This architecture allows React Native to provide native performance while writing JavaScript code.

Note: React Native is moving towards a new architecture with a feature called "the New Architecture" which includes Fabric (UI layer) and TurboModules (native modules) that will improve performance by reducing the reliance on the bridge.

Persisting State

In mobile apps, persisting state across app restarts is often necessary. React Native provides several options:

AsyncStorage

A simple, unencrypted, asynchronous, persistent, key-value storage system for React Native.

Using AsyncStorage 
import AsyncStorage from '@react-native-async-storage/async-storage';

// Storing data
const storeData = async (key, value) => {
  try {
    const jsonValue = JSON.stringify(value);
    await AsyncStorage.setItem(key, jsonValue);
  } catch (e) {
    console.error('Error saving data', e);
  }
};

// Reading data
const getData = async (key) => {
  try {
    const jsonValue = await AsyncStorage.getItem(key);
    return jsonValue != null ? JSON.parse(jsonValue) : null;
  } catch (e) {
    console.error('Error reading data', e);
    return null;
  }
};

MMKV

A high-performance, key-value storage framework developed by WeChat, offering better performance than AsyncStorage.

Using MMKV

Security Considerations

For sensitive data like authentication tokens or personal information, consider using encrypted storage solutions like react-native-encrypted-storage or expo-secure-store.

Using Secure Storage 
import * as SecureStore from 'expo-secure-store';

// Save securely
async function saveSecurely(key, value) {
  await SecureStore.setItemAsync(key, value);
}

// Get secure data
async function getSecureData(key) {
  return await SecureStore.getItemAsync(key);
}

// Example usage
await saveSecurely('authToken', 'my-secure-token-value');
const token = await getSecureData('authToken');

Best Practices and Tips

Performance Optimization

Use FlatList for Long Lists

For rendering long lists of data, always use FlatList or SectionList instead of map() with ScrollView. These components implement windowing, which renders only the items currently visible on screen.

FlatList Performance Tips
  • Use keyExtractor to provide a unique key for each item
  • Implement getItemLayout when possible to optimize rendering
  • Use removeClippedSubviews for large lists
  • Set appropriate initialNumToRender and maxToRenderPerBatch values
  • Memoize your renderItem function with useCallback

Memoize Components

Use React.memo, useMemo, and useCallback to prevent unnecessary re-renders.

Memoization Example 
import React, { useMemo, useCallback } from 'react';

// Memoize a component
const MemoizedComponent = React.memo(({ data }) => {
  // Component logic
});

// Inside a component
const ExpensiveComponent = ({ data, onSave }) => {
  // Memoize expensive calculations
  const processedData = useMemo(() => {
    return expensiveCalculation(data);
  }, [data]);
  
  // Memoize callbacks
  const handleSave = useCallback(() => {
    onSave(processedData);
  }, [onSave, processedData]);
  
  return (
    // Component JSX
  );
};

Optimize Images

Use appropriate image sizes and formats. Consider using libraries like react-native-fast-image for improved image loading performance.

Image Optimization Tips
  • Resize images to the actual dimensions needed
  • Use appropriate image formats (PNG for transparency, JPEG for photos)
  • Consider using WebP format for better compression
  • Implement progressive loading for large images
  • Use image caching libraries

Monitor JavaScript Thread

React Native runs your JavaScript code in a single thread. If this thread is blocked for too long, your app will become unresponsive.

Keeping the JS Thread Free
  • Move expensive calculations to web workers when possible
  • Break up long-running tasks into smaller chunks
  • Use InteractionManager to defer non-critical work until after animations
  • Profile your app with the Performance Monitor
