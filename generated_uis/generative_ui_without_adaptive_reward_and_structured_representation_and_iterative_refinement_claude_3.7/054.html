Introduction to React Native

Welcome! This guide is designed for web developers familiar with React who are looking to transition into mobile app development using React Native. We'll cover the fundamental differences, key concepts, and practical steps to get you started.

React Native lets you build mobile apps using JavaScript and React. It uses native components, giving your app a true native look and feel, unlike hybrid solutions that often rely on web views.

Why React Native?

  • Code Reusability: Share a significant portion of your codebase between iOS and Android.
  • Native Performance: Renders using native UI components, not web views.
  • Large Community: Benefit from the vast React ecosystem and community support.
  • Fast Development: Hot reloading and a familiar development experience speed up iteration.

React Native Ecosystem Overview

Understanding the tools and platforms around React Native is crucial for a smooth development experience.

Node.js and npm/Yarn

React Native development requires Node.js. You'll use npm or Yarn to manage project dependencies.


# Check Node.js version
node -v

# Check npm version
npm -v

# Check Yarn version (if installed)
yarn -v

Expo vs. React Native CLI

These are the two primary ways to set up and manage React Native projects.

Expo is a framework and platform for universal React applications. It provides a managed workflow that simplifies setup, development, and deployment, especially for beginners.

Pros:

  • Quick setup, no native build tools needed initially.
  • Easy testing on devices using the Expo Go app.
  • Access to a large library of APIs (camera, sensors, etc.).
  • Simplified builds and deployments.

Cons:

  • Limited access to native modules not included in the Expo SDK.
  • Requires "ejecting" to the bare workflow for full native control.

# Install Expo CLI
npm install -g expo-cli

# Create a new project
expo init my-expo-app

# Start the development server
cd my-expo-app
expo start

The React Native CLI requires more setup (Xcode/Android Studio) but gives you full control over the native code.

Pros:

  • Full control over native modules and configurations.
  • Suitable for complex apps requiring custom native code.

Cons:

  • More complex setup process.
  • Requires managing native build environments.

# Install React Native CLI
npm install -g react-native-cli

# Create a new project
react-native init my-native-app

# Run on iOS (requires Xcode)
cd my-native-app
react-native run-ios

# Run on Android (requires Android Studio)
react-native run-android

Core Components

Unlike web development where you use HTML elements like <div>, <p>, <img>, React Native uses its own set of built-in components that map directly to native UI elements.

Web HTML Element React Native Component Description
<div> <View> The most fundamental component for building UI. Acts as a container, supports layout with Flexbox, styling, and touch handling. Analogous to a div or section.
<p>, <h1>, <span> <Text> Used to display text. All text in React Native must be wrapped inside a <Text> component. Supports nesting for styling.
<img> <Image> Displays images. Supports network images, local assets, and base64. Requires specifying dimensions.
<button> <Button>, <TouchableOpacity>, <Pressable> <Button> is basic and platform-specific. <TouchableOpacity>/<Pressable> provide more customizable press feedback and are generally preferred.
<input type="text"> <TextInput> Allows users to input text. Highly configurable with props for keyboards, placeholders, validation, etc.
(Scrolling content) <ScrollView> A container that makes its content scrollable. Essential for content that might exceed screen dimensions.

You'll import these components from the react-native library:


import { View, Text, Image, TouchableOpacity, TextInput, ScrollView } from 'react-native';

// Example Usage
function MyComponent() {
  return (
    <ScrollView>
      <View style={{ padding: 20 }}>
        <Text style={{ fontSize: 24, fontWeight: 'bold' }}>Hello React Native!</Text>
        <Image
          source={{ uri: 'https://placehold.co/300x150?text=RN+Image' }}
          style={{ width: 300, height: 150, marginTop: 20 }}
        />
        <TextInput
          placeholder="Enter text..."
          style={{ height: 40, borderColor: 'gray', borderWidth: 1, marginTop: 20 }}
        />
        <TouchableOpacity onPress={() => alert('Button Pressed!')} style={{ backgroundColor: 'blue', padding: 10, marginTop: 20 }}>
          <Text style={{ color: 'white', textAlign: 'center' }}>Press Me</Text>
        </TouchableOpacity>
      </View>
    </ScrollView>
  );
}

Development Workflow

The React Native workflow involves writing JavaScript/React code, bundling it, and running it on simulators or physical devices.

Key Steps:

  1. Setup: Install Node.js, npm/Yarn, and either Expo CLI or React Native CLI with native development environments (Xcode for iOS, Android Studio for Android).
  2. Project Creation: Use expo init or react-native init.
  3. Writing Code: Develop components using JSX and JavaScript/TypeScript. Styling is done using JavaScript objects or StyleSheet.create.
  4. Running the App:
    • Expo: expo start, scan QR code with Expo Go app or run on simulator/emulator.
    • CLI: react-native run-ios or react-native run-android.
  5. Bundling: The Metro bundler compiles your JavaScript code and assets.
  6. Debugging: Use browser developer tools (Chrome/Safari), React Native Debugger, or VS Code debugging extensions.
  7. Testing: Test on simulators/emulators and physical devices.
  8. Building & Deploying:
    • Expo: Use expo build or expo publish.
    • CLI: Use native build tools (Xcode, Android Studio) to create IPA (iOS) or APK/AAB (Android) files.

Styling Differences

React Native uses JavaScript objects for styling, similar to inline styles in web React, but powered by native layout systems (Yoga, which implements Flexbox).


import { StyleSheet, View, Text } from 'react-native';

function StyledComponent() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Styled Text</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#f0f0f0',
  },
  title: {
    fontSize: 20,
    color: 'navy',
  },
});

Key differences from CSS:

  • Properties are camelCased (e.g., backgroundColor instead of background-color).
  • Units like px, em, rem are not used; values are unitless and represent density-independent pixels.
  • Shorthand properties (like margin, padding) are limited or behave differently; you often use explicit properties (marginTop, paddingHorizontal).
  • Inheritance works differently, especially for text styles.

State Management

Managing state in React Native apps is similar to React web apps, but the considerations for mobile (offline support, performance on limited devices, background tasks) can influence choices.

Comparison: Web vs. React Native State

Aspect Web React React Native
Basic Component State useState, useReducer (same) useState, useReducer (same)
Context API createContext, useContext (same) createContext, useContext (same)
Global State Libraries Redux, Zustand, MobX, Recoil, etc. Redux, Zustand, MobX, Recoil, etc. (most work cross-platform)
Persistence Local Storage, Session Storage, Cookies, IndexedDB AsyncStorage (React Native built-in), libraries like Redux Persist, Realm, SQLite
Offline Support Service Workers, Cache API Background tasks, NetInfo API, persistent storage solutions
UI Updates DOM manipulation (handled by React) Native UI updates via the bridge (handled by React Native)

Common Approaches in React Native:

  1. Component State: For local, component-specific state using useState or useReducer.
  2. Context API: For sharing state between components without prop drilling, suitable for themes, authentication status, etc.
  3. Global State Libraries: For complex applications with shared state across many components. Redux and Zustand are popular choices.
  4. Persistence: Use AsyncStorage for simple key-value storage or dedicated databases/libraries for more complex persistent data.

AsyncStorage is a simple, unencrypted, asynchronous, persistent key-value storage system for React Native. It's global to the app.


import AsyncStorage from '@react-native-async-storage/async-storage';

const storeData = async (key, value) => {
  try {
    await AsyncStorage.setItem(key, value);
    console.log('Data stored successfully');
  } catch (e) {
    // saving error
    console.error('Error storing data:', e);
  }
};

const getData = async (key) => {
  try {
    const value = await AsyncStorage.getItem(key);
    if (value !== null) {
      // value previously stored
      return value;
    }
  } catch (e) {
    // error reading value
    console.error('Error reading data:', e);
  }
  return null;
};

Remember that AsyncStorage is asynchronous, so you'll typically use async/await with it.

Best Practices

Following best practices helps build performant, maintainable, and scalable React Native applications.

  • Use StyleSheet.create: This optimizes styles by sending them over the bridge only once.
  • Optimize Images: Use appropriate formats (PNG, JPEG, WebP), sizes, and consider caching.
  • Manage Lists Efficiently: Use FlatList or SectionList for long lists to render only visible items, improving performance.
  • Handle Layout Carefully: Understand Flexbox and its nuances in React Native. Avoid complex nested views where possible.
  • Performance Monitoring: Use React Native's built-in performance monitor and debugging tools.
  • Navigation: Use a dedicated navigation library like React Navigation.
  • Platform-Specific Code: Use Platform module or .ios.js / .android.js extensions for platform-specific logic or UI.
  • Code Organization: Structure your project logically (e.g., by feature, by component type).
  • Error Handling: Implement robust error boundaries and logging.

Example: Using FlatList


import { FlatList, Text, View } from 'react-native';

const DATA = [
  { id: '1', title: 'First Item' },
  { id: '2', title: 'Second Item' },
  { id: '3', title: 'Third Item' },
  // ... many more items
];

const Item = ({ title }) => (
  <View style={{ padding: 20, borderBottomWidth: 1, borderColor: '#ccc' }}>
    <Text>{title}</Text>
  </View>
);

function MyFlatList() {
  return (
    <FlatList
      data={DATA}
      renderItem={({ item }) => <Item title={item.title} />}
      keyExtractor={item => item.id}
    />
  );
}

Key Concepts

A quick summary of important terms and concepts you'll encounter.

The bridge is the core mechanism that allows JavaScript code to communicate with the native platform (iOS/Android). JavaScript calls trigger native module executions, and native events are sent back to JavaScript.

Native Modules are platform-specific code (Objective-C/Swift for iOS, Java/Kotlin for Android) that can be invoked from JavaScript. This is how React Native accesses native APIs not available in JavaScript (e.g., Bluetooth, advanced camera features).

These are the UI building blocks provided by React Native (View, Text, etc.) that render directly to their native equivalents (e.g., <View> maps to UIView on iOS and <div>-like concepts on Android). This is different from web components which render to DOM elements.

Metro is the JavaScript bundler for React Native. It takes your JavaScript code and assets and bundles them for your mobile app. It supports features like hot reloading.
