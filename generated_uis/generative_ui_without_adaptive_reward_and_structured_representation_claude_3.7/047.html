cloud Introduction to Cloud Computing

Cloud computing is the on-demand delivery of IT resources—including computing power, storage, and databases—over the internet with pay-as-you-go pricing. It offers significant benefits like increased flexibility, scalability, and cost-efficiency compared to traditional on-premises infrastructure.

Cloud services are typically delivered via three main deployment models: Public Cloud (owned and operated by a third-party cloud service provider), Private Cloud (resources used exclusively by a single organization), and Hybrid Cloud (a combination of public and private clouds).

Key Takeaway

Cloud computing transforms IT from a capital expense to an operational expense, enabling businesses to innovate faster and scale globally.

category Core Cloud Service Categories

Cloud services are broadly categorized based on the level of management provided by the cloud provider versus the customer. The three main categories are Infrastructure as a Service (IaaS), Platform as a Service (PaaS), and Software as a Service (SaaS).

IaaS provides the fundamental building blocks of cloud IT. It gives you access to networking features, computers (virtual or on dedicated hardware), and data storage space. IaaS provides the highest level of flexibility and management control over your IT resources and is most similar to existing IT resources that many IT departments and developers are familiar with.

With IaaS, you manage the operating systems, data, applications, and middleware, while the cloud provider manages the virtualization, servers, storage, and networking.

PaaS removes the need for you to manage underlying infrastructure (hardware and operating systems) and enables you to focus on the deployment and management of your applications. PaaS provides the building blocks for creating modern applications, such as managed databases, web servers, and development tools.

With PaaS, you manage the applications and data, while the cloud provider manages the runtime, middleware, operating system, virtualization, servers, storage, and networking.

SaaS provides a complete product that is run and managed by the service provider. In most cases, people referring to SaaS are referring to end-user applications. With a SaaS offering, you don't have to think about how the service is maintained or how the underlying infrastructure is managed; you just need to think about how you will use that particular software.

With SaaS, the cloud provider manages everything: the application, data, runtime, middleware, operating system, virtualization, servers, storage, and networking.

architecture Common Architectural Patterns

Choosing the right architectural pattern is crucial for building scalable, resilient, and maintainable cloud applications. Here are some common patterns:

In a monolithic architecture, the entire application is built as a single, indivisible unit. All components (UI, business logic, data access) are tightly coupled and run within a single process. This is often simpler to develop and deploy initially but can become complex and difficult to scale or update as the application grows.

Microservices architecture structures an application as a collection of small, autonomous services, modeled around a business domain. Each service is self-contained, can be developed, deployed, and scaled independently, and communicates with other services, typically over a network.

Serverless architecture allows you to build and run applications and services without thinking about servers. The cloud provider manages the infrastructure, and you only pay for the compute time you consume. This often involves using Function-as-a-Service (FaaS) like AWS Lambda or Azure Functions, triggered by events.

A serverless data processing pipeline could involve uploading a file to cloud storage (triggering an event), which invokes a serverless function to process the file, extract data, and store it in a database or another storage service. book Glossary of Key Terms

IaaS
Infrastructure as a Service. A cloud service model providing virtualized computing resources over the internet.
PaaS
Platform as a Service. A cloud service model providing a platform for developing, running, and managing applications without managing infrastructure.
SaaS
Software as a Service. A cloud service model providing software applications over the internet on a subscription basis.
Virtual Machine (VM)
An emulation of a computer system. VMs provide the functionality of a physical computer, allowing you to run operating systems and applications.
Container
A standard unit of software that packages up code and all its dependencies so the application runs quickly and reliably from one computing environment to another.
Serverless Function
A small, single-purpose piece of code that runs in response to an event, without requiring you to provision or manage servers.
Scalability
The ability of a system to handle a growing amount of work by adding resources.
Resilience
The ability of a system to recover from failures and continue to function.
API Gateway
A server that acts as an entry point for clients to access microservices.
Message Queue
A component used for asynchronous communication between different parts of an application or different services.
